### #13 身份 IP 来源分级（机场轮换 / 国内固定 / 国外固定）（✅ 已完成）

> 背景：小红书/抖音等国内严风控平台对 IP 有三条硬要求，机场节点三条全踩雷：①要大陆住宅/4G IP，机场是海外 IDC；
> ②要独享稳定，机场多账号共享一节点（关联风控）；③登录后 IP 不能跳，而 #11 会自动换节点（触发异地登录风控）。
> 所以这类平台**不能用统一分配的机场节点**，需要专属、固定、不轮换的住宅/静态 IP。

**核心抽象**：区分的不是「国内/国外」，而是**身份的「IP 来源类型」**；国内/国外只是固定 IP 身份的 `region` 属性。

| 身份类型 | IP 来源 | 是否轮换 | 给谁用 |
|---|---|---|---|
| 邮箱身份（现有） | 机场池节点（海外 IDC） | 自动轮换 ✓ | Google 登录的海外平台 |
| 固定 IP 身份 · 国内 | 用户填的国内住宅/4G 代理 | 钉死 ✗ | 小红书/抖音/微博 |
| 固定 IP 身份 · 国外 | 用户填的海外静态代理 | 钉死 ✗ | 需要稳定海外 IP 的平台 |

**与现有代码的契合点**：#11 轮换只动 `personas.node_name IS NOT NULL`；固定 IP 身份不分配机场节点（`node_name=NULL`），
直接用 `set_profile_proxy` 给 profile 绑用户填的代理 → **天然被排除在轮换之外**，不走 mihomo 机场那套。

**数据模型（拟）**：
- `personas` 增 `ip_mode`（`airport` / `fixed`）+ `fixed_proxy`（socks5/http）+ `region`（cn/海外）。创建固定身份时不取节点池、改让用户填代理。
- `PlatformMeta` 增 `ip_policy`（`shared_overseas` / `residential_cn` / `static_overseas`），用于把平台路由到对应身份类型：
  - 邮箱身份「开通账号」只列 `shared_overseas` 的 Google 平台；
  - 国内固定身份「加账号」只列 `residential_cn` 平台（小红书…）；互不串台。

**已拍板的决策（按 Claude 建议，后续可调）**：
1. **固定 IP 身份限「一身份 = 一 IP = 一个号」**（尤其国内，避免同 IP 多号关联）。
2. **国内/国外底层是同一类「固定 IP 身份」**（带 `region` 字段），UI 上按 region 分组/分 tab 展示。
3. **代理来源先「用户手填单条」**，后续再考虑接「住宅代理池」（类似机场订阅，但走住宅服务商 API）。

**产品预期提示**：即便 IP 完美，小红书 web 自动化仍可能被设备指纹/行为风控；Class B 平台需标注「自备住宅/4G 代理、封号风险较高、建议低频」。

**实现（✅ 已完成）**：
- 后端（`src-tauri/src/lib.rs`）：
  - 迁移：`personas` 加列 `ip_mode`(默认 `airport`) / `fixed_proxy` / `region`（已建库自动 ALTER，向后兼容）。
  - `platform_ip_policy(platform)` → `shared_overseas` / `residential_cn` / `static_overseas`（**按各平台真实风控严格度逐个标注**，见下表）；
    `PlatformCatalogItem` 增 `ip_policy` 字段返回。

  **平台 IP 策略分布（逐平台标注）**：
  | 策略 | 含义 | 平台 |
  |---|---|---|
  | `residential_cn` | 需国内住宅/4G 固定 IP、不可轮换 | 小红书、微博（只留确定需要固定 IP 的强风控平台） |
  | `static_overseas` | 需稳定独享海外固定 IP、不可轮换 | Twitter/X、Reddit、LinkedIn、Facebook、VK、Naver Blog |
  | `shared_overseas` | 对 IP 宽松，机场共享轮换即可 | GitHub、DEV.to、Hashnode、Hacker News、Qiita、Zenn、Habr、V2EX、SegmentFault、Product Hunt、BetaList、AlternativeTo、Indie Hackers、Medium、note、Telegram、**知乎、即刻、CSDN、少数派、开源中国**（手机号登录≠IP严，挂 Gmail 身份走机场 IP 即可） |
