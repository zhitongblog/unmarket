## 完成记录

### #1 各平台标记登录方式（✅ 已完成）

- 后端（`src-tauri/src/lib.rs`）：
  - `PlatformCatalogItem` 新增 `login_method` 字段（`google` / `phone` / `password`）。
  - 新增 `platform_login_method()`：以 `PlatformConfig.google_oauth` 为单一事实来源推导 `google`；
    手机验证码主导平台（知乎/微博/少数派/即刻/小红书/CSDN/Telegram）标 `phone`；其余标 `password`。
  - `persona_platform_catalog` 一并返回 `login_method`。
- 前端（`src/tauri-frontend/app.ts`）：
  - `CatalogItem` 接口加 `login_method`；「开通账号」选择器每张平台卡片新增登录方式徽章
    （🔵 Google 登录 / 📱 手机号 / 🔑 账号密码），与开通模式（自动/需手动）徽章并列，便于区分流程。

### #2 国际化检查并修正（✅ 已完成）

- 新增 `tf(key, vars)` 翻译插值helper，支持 `{email}` / `{n}` 等占位符。
- 「加账号 / 开通账号」相关动态文案全面接入 i18n（中英双语）：
  - 平台选择器弹窗 `pickProvisionPlatforms`：标题、说明、场景分组标题、已开通/自动/需手动徽章、
    登录方式徽章、底部按钮（全选可开通/取消/应用更改）。
  - 身份流程 `createPersonaPrompt` / `personaProvisionAll` / `deletePersonaAcct`：
    新建/创建中/已创建/删除确认/各类失败提示等 Toast。
  - `saveAccount` / `deleteAccount`：必填校验、保存失败、删除确认、删除失败提示。
- 静态「添加账号」弹窗（`dist/tauri/index.html` 的 `#modalAddAccount`）补齐 `data-i18n` /
  `data-i18n-placeholder`（标题、平台/用户名/密码标签与占位符、取消/保存按钮）。
- 新增 i18n key 分组：`provision.*`、`login.*`、`scene.*`、`persona.*`、`accounts.*`（弹窗相关），
  zh / en 两套完整对齐。

### #3 登录 Gmail 复用已打开的 Unzoo 浏览器（✅ 已完成）

- 后端（`src-tauri/src/lib.rs`）：
  - 新增 `navigate_existing_profile_tab(profile_path, url)`：GET `/tabs` 找该 profile 已打开的标签，
    优先复用 `about:blank` 空白页（其次激活页、再其次第一个）→ POST `/navigate` 导航；找不到返回 Err。
  - `persona_open_gmail_login` 改为：① 先试复用已打开窗口；② 失败再走原 `open_profile_window` 新开。

### #4 加账号按登录方式分区 + 录入凭据（✅ 已完成）

- 前端（`src/tauri-frontend/app.ts`）：
  - 新增 `pickAddAccounts(email, candidates)`：按「📱 手机号」「🔑 账号密码」两区块渲染，手机号平台一个手机号输入框、
    帐密平台账号 + 密码两输入框；底部「添加 (n)」按填了主字段的数量实时更新。
  - 新增 `personaAddAccounts(id,email)`：拉 `persona_platform_catalog`，过滤 `login_method ∈ {phone,password}` 且 `!provisioned`，
    逐个 `add_account` + `set_account_persona` 归属到该邮箱。已添加（provisioned）的平台不再出现。
  - 邮箱面板「+ 加账号」按钮改为调用 `personaAddAccounts`（不再打开旧的静态 `modalAddAccount`）。
  - 新增 i18n `addacct.*`（zh/en）。

### #5 开通账号选择页隐藏已开通平台（✅ 已完成）

- `personaProvisionAll` 只把 `login_method === 'google' && !provisioned` 的平台传入 `pickProvisionPlatforms`；
  全部为未开通项 → 勾选即开通，移除已开通账号改由账号卡片删除完成（选择器不再承担"取消勾选删账号"）。
  无可开通项时提示 `provision.allDone`。

### #6 邮箱旁展示连接的 Unzoo profile 名称（✅ 已完成）

- 新增 `profileNameOf(persona)`：用 `availableProfiles`（`get_available_browser_profiles` 返回的 `{id,name}`）
  把 `persona.profile_id`（profile 文件夹名）映射为可读名称，展示在邮箱面板 meta 行 `🖥️ 浏览器: <名称>`。
- **修复（二次）**：原 `get_available_browser_profiles` 用 `path.split('\\').last()` 取 profile id，
  在 macOS（路径用 `/`、无 `\`）下会返回**整条路径**而非文件夹名，导致 `id` 与 `persona.profile_id`（文件夹名）
  对不上、只能回退显示难看的文件夹名。改为 `rsplit(|c| c=='/'||c=='\\')` 取末段文件夹名，
  与 `personas/accounts.profile_id` 统一；同时修好了账号卡片里 profile 绑定下拉的选中高亮。
  `profileNameOf` 同步加健壮匹配（按末段比对），匹配不到则不展示该项（不再回退文件夹名）。

### #9 邮箱切换条另起一行与平台编组（✅ 已完成）

- 新增 `.email-group` 卡片把「邮箱 + 平台」编成一组：
  - group head：邮箱 tab 切换条（`.email-tab` 胶囊样式，选中高亮）与「收起/展开平台」按钮同一行；
